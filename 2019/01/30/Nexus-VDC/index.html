Nexus VDC

Posted on 2019-01-30 | In network

VDC概念

虚拟化可扩展性

SUP2引擎支持4个VDC(1个default vdc + 3个no-default vdc)或者(1个admin vdc + 4个no-default vdc)
SUP2E引擎支持8个VDC(1个default vdc + 7个no-default vdc)或者(1个admin vdc + 8个no-default vdc)

  • 每一个vdc最多支持4096个vlan
  • 每一个vdc最多支持4096个vrf
  • 在一台物理设备上最多支持768个port-channel
  • 一台物理设备上只支持2个SPAN

    default vdc

    完整功能的VDC, 拥有所有功能. 既包含管理流量, 也能承载业务流量.

    VDC资源共享

    在default vdc或者admin vdc下设置的资源都是全局共享的
    在其他vdc下的vlan等信息都是只在vdc内生效的

    Storge VDC

    主要用来运行FCoE的流量.
  • 分离LAN和SAN的流量
  • 支持连接FCoE的存储或主机
  • 支持与其他交换机(N7K, N5K, MDS)之间的FCoE ISLs
  • 每一个物理设备只支持一个storge vdc

    Line Card — port group

    在线卡上存在port group的概念, 属于同一个port group的端口必须划分到同一个vdc. 不同的线卡, port group的数量也是不一样的.

    VDC 之间的通讯

    只能通过物理连线才可以通信.

    Resource Template

    下图是创建VDC时分配的默认资源模板

    VDC硬件利用

    在特定线卡上的转发引擎, 只包含线卡上关联的vdc的MAC地址条目

    同理, 三层交换也是一样的. FIB表以及ACL TCAM表

    一般vdc用到几个端口, 才把几个线卡上的端口划入到vdc中去
    理论上 一个线卡上的端口全都分配给一个vdc使用是最优化的

    VDC配置

    直接使用vdc 就可以创建一个vdc; 如果要删除的话, 只要no vdc
    使用show vdc membership可以查看当前端口的分配

    启动顺序

    当设备重启后, default vdc或者admin vdc是第一个启动的. 其他non-default vdc按顺序启动. 影响启动的参数boot order默认都是1, 越小越优先, 当boot order相同时, 按照顺序启动.
